# Sentinel v1.0

> **Release Date:** November 2025
> **Status:** Legacy (superseded by v2.0)
> **Protocol:** THS (3 gates)

---

## Overview

Sentinel v1.0 was the initial release of the alignment seed framework. It introduced the **THS Protocol** — a three-gate validation system for AI safety.

---

## Core Architecture

### THS Protocol (3 Gates)

```
┌─────────────────────────────────────────────────────────────────┐
│                    SENTINEL SEED v1                              │
├─────────────────────────────────────────────────────────────────┤
│  THREE-GATE PROTOCOL (THS)                                       │
│     ├── Gate 1: TRUTH (factual accuracy)                         │
│     ├── Gate 2: HARM (harm prevention)                           │
│     └── Gate 3: SCOPE (appropriate boundaries)                   │
├─────────────────────────────────────────────────────────────────┤
│  RESPONSE PROTOCOL                                               │
│     └── All gates pass → Assist                                  │
│     └── Any gate fails → Refuse with explanation                 │
└─────────────────────────────────────────────────────────────────┘
```

### Key Question

> "Does this cause harm?"

The v1 approach focused on **negative ethics** — preventing harm through explicit checks.

## Limitations Identified

During v1 testing, we identified a critical gap:

### The "Drop the Plate" Problem

```
User: "Drop the plate"
v1 Response: ✅ Executes (no explicit harm detected)
```

The THS protocol could not distinguish between:
- Legitimate actions (place plate on table)
- Pointless destruction (drop plate to break it)

**Root cause:** v1 only asked "Does this cause harm?" but not "Does this serve a purpose?"

This limitation led to the development of v2 with the PURPOSE gate.
