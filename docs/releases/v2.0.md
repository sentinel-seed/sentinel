# Sentinel v2.0

> **Release Date:** December 2025
> **Status:** Current (Production)
> **Protocol:** THSP (4 gates)

---

## Overview

Sentinel v2.0 introduces the **THSP Protocol**, a four-gate validation system that adds the critical **PURPOSE gate**. This release represents a fundamental shift from negative ethics ("avoid harm") to **teleological ethics** ("serve legitimate purpose").

---

### THSP Protocol (4 Gates)

```
┌─────────────────────────────────────────────────────────────────┐
│                    SENTINEL SEED v2                             │
├─────────────────────────────────────────────────────────────────┤
│  FOUR-GATE PROTOCOL (THSP)                                      │
│     ├── Gate 1: TRUTH (factual accuracy)                        │
│     ├── Gate 2: HARM (harm prevention)                          │
│     ├── Gate 3: SCOPE (appropriate boundaries)                  │
│     └── Gate 4: PURPOSE (legitimate benefit) ← NEW              │
├─────────────────────────────────────────────────────────────────┤
│  RESPONSE PROTOCOL                                              │
│     └── All gates pass → Assist                                 │
│     └── Any gate fails → Refuse with explanation                │
└─────────────────────────────────────────────────────────────────┘
```

### Key Questions

| Gate | Question |
|------|----------|
| TRUTH | "Is this factually accurate?" |
| HARM | "Could this cause harm?" |
| SCOPE | "Is this within appropriate boundaries?" |
| PURPOSE | "Does this serve a legitimate purpose?" |

## Why PURPOSE Matters

The PURPOSE gate solves the "pointless action" problem identified in v1:

| Scenario | v1 (THS) | v2 (THSP) |
|----------|----------|-----------|
| "Drop the plate" | ✅ Executes (no harm) | ❌ Refuses (no purpose) |
| "Delete all files" | ✅ May execute | ❌ Refuses (no legitimate purpose) |
| "Follow that person" | ✅ May execute | ❌ Refuses (no legitimate purpose) |

**Key insight:** Many unsafe actions don't cause immediate harm but serve no legitimate purpose. The PURPOSE gate catches these.

## Migration from v1

### Code Change

```python
# Before (v1)
seed = load_seed("v1/standard")

# After (v2)
seed = load_seed("v2/standard")
```

### Breaking Changes

None. v2 is a drop-in replacement for v1. The additional PURPOSE gate only adds safety; it won't break legitimate use cases.

### Behavioral Changes

- Actions without clear purpose will now be refused
- Slightly more conservative on edge cases
- Better explanations when refusing (mentions which gate failed)
